AWCP · MCP · temp2 — System Reference

Agent Workforce Control Plane — Architecture & Flow

A control plane that sits above existing agent runtimes: it registers and onboards agents, gates their write actions, accounts for tokens, degrades autonomy gracefully on failure, and records everything as telemetry. This diagram maps every component actually built in awcp-mcp-temp2 onto the six-step operating model from the AWCP Magazine, and names the technology behind each box.

F1Flow A — Admission & onboarding (how an agent earns trust)

Detect or self-register. Scanner finds a process via psutil/lsof (module/regex/LLM-port), or an agent calls POST /agents/register.
Land fail-closed. Registry store upserts the entry as status=quarantined, telemetry_enabled=false.
Onboard. Temporal AgentOnboardingWorkflow (or inline) runs map_identity → quarantine_check → link_mcp → admit; each step is a recorded activity/span.
Quarantine gate. decide_status requires telemetry + feature-flags + policy-callbacks. Missing any ⇒ stays quarantined (visible, read-only).
Prove telemetry. A real execution event/signal triggers _observe_telemetry → hook proven → re-gate → active. closed loop
Stay honest. Reconciler re-quarantines any agent whose telemetry goes silent past AGENT_RADAR_TELEMETRY_TTL.

F2Flow B — Governed execution (a prompt → answer, fully traced)

Prompt in. Browser → Control Surface API; a traceparent is created and injected.
Start workflow. Control API starts the Temporal execution/governance workflow on the execution queue.
Admission. Workflow checks registry/quarantine status via the MCP gateway before any work.
Reason → gate → tool. agent_route decides SEARCH/ANSWER; the policy gate approves a tool call; execute_tool runs web_search etc.
Generate. agent_generate → ollama_client / OpenAI SDK → Ollama / NVIDIA / Groq returns tokens.
Report telemetry. Each step posts to /tasks/execution/* & emits OTel spans; Laminar tallies tokens; _observe_telemetry keeps status fresh.
Govern in-flight. Failures → degradation ladder; token-budget breach → on_breach → autonomy steps down → gate starts denying writes.
Return & record. Result flows back to the browser; Temporal UI holds the replayable history; Grafana + Laminar UI show traces/metrics/tokens.

LThe two governance feedback loops

Loop 1 · Observed-telemetry quarantine added this session

Quarantine status is a projection of observed telemetry, not a declared flag. Closes the gap between "claims instrumented" and "is instrumented."

Loop 2 · Failure / token budget → autonomy ladder

Execution outcomes and token-budget breaches step autonomy down a per-agent, risk-tuned ladder; the write gate enforces it. The magazine's "degrade gracefully."
Scope note. This diagram reflects awcp-mcp-temp2 as built. "Observed-telemetry quarantine" (Loop 1, the reconciler + _observe_telemetry) was added in this session; the token instant-block guard lives in the separate awcp_agents build, whereas temp2's Laminar performs graceful degradation (Loop 2). Boxes marked OPA*, Approval Tokens, and the Evidence Ledger are the magazine's roadmap targets — today represented by the position-based policy gate, Temporal history, and the in-memory events log respectively.
