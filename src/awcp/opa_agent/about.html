🛡️ OPA Agent hidden · background · SLM

SLM-reasoned tool-call policy decision point for the AWCP control plane.

A small language model reasons about each tool call and assigns its risk tier — low · medium · high · severe. The agent then decides allow / block (block when high/severe) and records a structured JSON per question, so the Radar can draw a tier bar for every tool call the five worker agents make. The block decision runs through OPA / Rego when configured, with a fail-secure fallback.
GET /healthservice status + tier vocabulary + SLM info
GET /tierstiers, block set, per-tool SLM tiers, recent calls (Radar feed)
GET /toolstiers, block set, per-tool tier map (read-only)
POST /evaluatereason a tool call's tier · decide · record JSON
GET /decisions/{task_id}the per-question tool-risk JSON

Not a worker agent — it never appears in /agents or the radar's agent table.
