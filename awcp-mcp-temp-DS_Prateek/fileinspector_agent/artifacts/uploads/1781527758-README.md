# AWCP — Phase 0: Local Development Foundation

This is the substrate every later phase builds on. It brings up a durable
workflow engine, a policy decision server, a telemetry pipeline, and the
databases behind them, all on one Docker network so future services can
reach them by name.

There is **no AWCP business logic here yet.** Phase 0 is done when all five
dependencies are healthy and you can watch a trace flow end-to-end.

## What's in the box

| Service          | Purpose                                              | URL / Port                  |
|------------------|------------------------------------------------------|-----------------------------|
| `postgres`       | Temporal persistence **and** the `awcp` app database | `localhost:5432`            |
| `temporal`       | Durable workflow engine (gRPC frontend)              | `localhost:7233`            |
| `temporal-ui`    | Inspect workflows in a browser                       | http://localhost:8080       |
| `opa`            | Policy decision server                               | http://localhost:8181       |
| `otel-collector` | Receives OTLP, fans out to Jaeger + debug log        | gRPC `:4317`, HTTP `:4318`  |
| `jaeger`         | Trace backend + UI (dev only — not the ledger)       | http://localhost:16686      |

## Prerequisites

- Docker Engine + Docker Compose v2 (`docker compose version` should print v2.x)
- ~6 GB RAM free for the stack
- `curl` and `bash` on the host (for the health script)
- Optional but recommended: the Temporal CLI on your host (`brew install temporal`)
  so you can run `temporal workflow list --address localhost:7233` interactively

## Quickstart

```bash
cd awcp

# 1. Validate every image tag in .env resolves (catches a bad pin immediately).
make pull            # == docker compose pull

# 2. Bring the foundation up.
make up              # == docker compose up -d

# 3. Wait ~30s for Temporal's schema setup, then check health.
make health
```

A clean run ends with:

```
  [ OK ]  Postgres (temporal db)
  [ OK ]  Postgres (awcp db)
  [ OK ]  Temporal cluster
  [ OK ]  Temporal UI
  [ OK ]  OPA server
  [ OK ]  OPA policy loaded
  [ OK ]  OTel collector
  [ OK ]  Jaeger UI
Passed: 8   Failed: 0
Foundation is up. You're ready for Phase 1.
```

## Validation — prove each piece works

**Temporal.** Open http://localhost:8080. You should see the Temporal UI with a
`default` namespace and an empty workflow list. (Phase 1 adds a dedicated
`awcp` namespace.)

**OPA.** Query the smoke-test rule and the default-deny posture:

```bash
curl -s localhost:8181/v1/data/awcp/system/ready   # -> {"result":true}
curl -s localhost:8181/v1/data/awcp/system/allow   # -> {"result":false}
```

The `allow` default of `false` is the whole control plane's security posture in
miniature: nothing is permitted unless a policy explicitly says so. Phase 4
replaces this stub with real write-gating logic.

**Telemetry end-to-end.** Emit test traces and watch them arrive:

```bash
make trace
```

Then open Jaeger at http://localhost:16686, pick the `awcp-smoke-test` service,
and click "Find Traces" — you should see 5 traces. You can also confirm the
collector saw them: `docker compose logs otel-collector | grep -i traces`.

> `make trace` joins the compose network (`awcp_awcp`) to reach the collector by
> name. If your network is named differently, check `docker network ls | grep awcp`
> and adjust the `--network` flag in the Makefile's `trace` target.

## Troubleshooting

- **A `pull` step 404s on a tag.** The pins in `.env` drifted. Open the image's
  Docker Hub tags page (links are in `.env`) and bump to the current tag. Keep
  `TEMPORAL_VERSION` and `TEMPORAL_UI_VERSION` compatible and bump them together.
- **Temporal health fails for the first ~30s.** Expected — `auto-setup` is
  creating its schema. Re-run `make health` after half a minute.
- **Apple Silicon / ARM.** All images here publish arm64 variants; no extra
  flags needed on recent Docker Desktop.
- **Port already in use.** Something else owns 5432/7233/8080/8181/16686. Stop it
  or remap the host side of the port in `docker-compose.yml`.
- **Reset to a clean slate.** `make clean` tears everything down and wipes the
  Postgres volume, so the next `up` re-runs `init-awcp-db.sql` and Temporal
  re-provisions its schema.

## How this connects to Phase 1

Phase 1 (the Agent Registry) plugs into exactly three seams established here:

1. It connects to the `awcp` Postgres database (created by `init-awcp-db.sql`)
   to store agent identity, owners, declared write scopes, feature flags, and
   policy-callback points.
2. It registers a dedicated `awcp` Temporal namespace and the custom Search
   Attributes that later phases query on (owner, autonomy mode, quarantine state).
3. It emits its first spans to the collector at `otel-collector:4317`, so the
   moment you register an agent you can see the call in Jaeger — the same
   telemetry path that becomes the Evidence Ledger in Phase 5.
