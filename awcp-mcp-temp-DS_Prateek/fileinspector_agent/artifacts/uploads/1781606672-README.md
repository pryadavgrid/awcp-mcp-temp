# Agent Workforce Control Plane (AWCP)

> A **governance/control layer that sits *above* existing agent runtimes** — it does not
> replace them. It registers agents, gates state-changing (write) actions behind policy and
> expiring approval tokens, reduces autonomy gracefully when failures rise, and replays/recovers
> failed branches from an immutable evidence ledger.
>
> Source spec: [`docs/magazine/Agent-Workforce-Control-Plane-Magazine.html`](docs/magazine/Agent-Workforce-Control-Plane-Magazine.html)

## 1. What this project is (from the magazine)

**Buyer:** Head of Agent Governance / VP, Agent Operations.
**Problem:** Teams run autonomous agents against real business systems (CRM, billing, support).
When agents fail, they either crash hard or keep writing bad state. There is no safe middle ground
and no auditable record to recover from.

**Core idea — 4 capabilities:**

| Capability | What it means |
|---|---|
| **Policy-gated writes** | Every state-changing tool/API call is evaluated against policy. High-risk calls pause until a narrow, *expiring* approval token is issued for that branch. |
| **Graceful autonomy reduction** | On failure signals the plane *first* raises trace sampling, *then* tightens retry/concurrency, *then* pins a safer model, *then* drops to recommendation-only, and only *then* hard-stops. |
| **Replayable recovery** | An evidence ledger stores step history, context hashes, degradation state, and safe resume points so a failed branch resumes without restarting the whole workflow. |
| **Lightweight onboarding** | Under-instrumented agents are registered but *quarantined* (write-blocked) until telemetry, feature-flag, and policy callbacks are merged. A patch generator proposes the missing hooks. |

**Evidence boundary (the contract the whole system enforces):**
- *Write-capable* = any call that creates, mutates, deploys, approves, messages, or changes state.
- *Human approval required* = new write scopes, cross-system writes, rollback/restore, policy exceptions, elevated autonomy.
- *Conservative mode* = read-only analysis, more tracing, recommendation-only continuations, evidence packets.
- *Evidence* = workflow identity, actor, policy result, approval token, tool call, context-snapshot hash, degradation state, replay trace, rollback pointer.

**90-day proof slice (the MVP):** connect **one** existing workflow → route its writes through
expiring approval tokens → keep it alive through progressive autonomy reduction → replay every
failure from the ledger.

## 2. The three signature scenarios (acceptance tests)

| Scenario | Trigger | Control-plane response | Safe outcome |
|---|---|---|---|
| **A — Graceful Degradation** | A branch breaches its failure budget (3 failed writes, rising latency, stale handoff context) | Increase trace capture → tighten retry/concurrency → pin safer model → drop to recommendation-only | Workflow stays alive, no new writes, replayable checkpoint + narrowed plan + evidence |
| **B — Approval Tokens** | Agent proposes a cross-system bulk write (CRM + billing) needing temp privilege elevation | Scope the write set + risk tier → bundle tool plan, context diff, rollback pointer, expiry → operator approves/denies/escalates | Single-use, branch-scoped, time-boxed token; all else blocked + logged + replayable |
| **C — Onboarding & Quarantine** | New under-instrumented agent appears via a runtime adapter (no traces, no flag callbacks) | Register owner/runtime/scope → quarantine (block governed writes) → generate instrumentation patch | Agent in inventory immediately, leaves quarantine only after hooks merged + observed |

These three map directly to `tests/scenarios/` and `fixtures/`.

## 4. Step-by-step build plan

The plan is sequenced so you always have a **runnable, demoable slice**. Each phase ends with a
working artifact, not just code.

### Phase 0 — Foundation & migration (week 1)
- [ ] Initialize repo: `git init`, monorepo tooling (pnpm workspaces for TS, `uv`/`poetry` for Python), `.env.example`, `Makefile`.
- [ ] Stand up local stack in `docker-compose.yml`: **Temporal**, **MinIO** (S3 ledger), **OPA**, **OTel collector** + Grafana/Tempo/Loki/Prometheus.
- [ ] Define the **core data contracts** in `packages/core-models`: `WorkflowIdentity`, `EvidenceEntry`, `ApprovalToken`, `DegradationState`, `WriteScope`. Everything else depends on these.
- [ ] **Migrate** proven code from `../Capstone_AWCP_Solution_Portfolio`: Cognitive Shield → `packages/context-engine`, Temporal backend → `services/workflow-engine`, OPA policies → `policies/`, MinIO ledger → `services/evidence-ledger`.
- **Exit:** `docker compose up` brings up Temporal + OPA + MinIO + OTel; models import cleanly.

### Phase 1 — Ingestion & registry (week 2)
- [ ] `adapters/rest` + one event consumer: receive a workflow event, normalize to `WorkflowIdentity`.
- [ ] `services/intake-proxy`: assign workflow ID, map owner, open a governed branch, emit first evidence entry.
- [ ] `services/agent-registry`: register agent (owner, runtime, declared write scopes, flags, callbacks); **quarantine** any agent missing required hooks.
- **Exit:** posting a sample event registers an agent and lands a ledger entry; under-instrumented agents land in quarantine.

### Phase 2 — Policy-gated writes (week 3) → **Scenario B**
- [ ] `policies/write_scopes.rego` + `approval.rego`: classify each tool call as auto-authorized / token-required / human-required by risk tier.
- [ ] `services/approval-gate`: on token-required calls, bundle {tool plan, context diff, rollback pointer, expiry} and **pause durably** (Temporal signal) until decision.
- [ ] Issue **narrow, single-use, time-boxed, branch-scoped** tokens; block + log everything else.
- **Exit:** `tests/scenarios/scenario_b_approval` passes — cross-system write pauses, operator grants a 15-min branch token, write executes, all attempts in ledger.

### Phase 3 — Graceful degradation (week 4) → **Scenario A**
- [ ] `services/degradation-engine`: failure-budget + stale-context + latency signals → the ladder: `raise sampling → tighten limits → safer profile → recommendation-only → hard stop`. Central defaults + per-workflow override ladders.
- [ ] `services/llm-gateway`: safer-profile routing (pin a safer model on degrade).
- [ ] `packages/context-engine`: context hashing + governing-slice assembly + stale detection (from Cognitive Shield).
- **Exit:** `tests/scenarios/scenario_a_degradation` passes — a budget breach narrows autonomy step-by-step, service stays alive, writes stop, each step is evidenced.

### Phase 4 — Replay & recovery (week 5)
- [ ] `services/evidence-ledger`: immutable append, chain-of-custody, context-hash index, resume pointers.
- [ ] `services/handoff-coordinator` + `context-graph`: branch isolation, checkpoints, safe resume from `checkpoint:t+NN`.
- [ ] `packages/context-engine` RLM summarizer: fold large traces into replay summaries.
- **Exit:** kill a workflow mid-branch, then resume *only* the failed branch from its checkpoint with full evidence — no full restart.

### Phase 5 — Operator UI (week 6) → **User Flow + Section 07**
- [ ] `ui/control-center`: risk-first KPIs (degraded workflows, pending tokens, write-blocked agents), workflow queue, tool/action gate.
- [ ] `ui/recovery-workbench`: recovery ladder, replay evidence, policy telemetry.
- [ ] `ui/quarantine-exit`: onboarding readiness checklist + action set.
- **Exit:** operator can review an alert → inspect replay → approve/narrow → resume/escalate, end to end.

### Phase 6 — Onboarding & patch generation (week 7) → **Scenario C**
- [ ] `tools/patch-generator`: wrap Codex CLI / Claude Code to propose telemetry + flag + policy-hook patches for quarantined agents.
- [ ] Quarantine-exit gate: agent leaves quarantine only after patch merged **and** hooks observed in execution.
- **Exit:** `tests/scenarios/scenario_c_quarantine` passes — new agent registered, quarantined, patched, promoted.

### Phase 7 — Hardening & demo (week 8+)
- [ ] End-to-end run of all three scenarios from one `make demo`.
- [ ] OTel traces visible in Grafana/Tempo; ledger queryable.
- [ ] Docs: `docs/architecture.md`, sequence + data-flow diagrams, runbook.
- **Exit:** clean `docker compose up` + `make demo` reproduces the magazine's claims with **observed** (not illustrative) metrics.

## 6. Tech stack (recommended)

- **Orchestration:** Temporal (durable DAGs, replay, long approval waits)
- **Policy:** OPA / Rego
- **Evidence:** OpenTelemetry + object store (MinIO local → S3)
- **Execution:** CodeAct + Modal (Python subprocess locally); LLM Gateway over Anthropic/Claude + OpenAI
- **Context:** context-graph + hashing + RLM compression (your Cognitive Shield); Letta for long-term memory
- **Flags:** OpenFeature / Flipt (autonomy modes, write-scope switches)
- **UI:** Next.js / React risk-first dashboard
- **Patches:** Codex CLI / Claude Code CLI
