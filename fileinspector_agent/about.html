🗂 File Inspector Worker

A universal file-inspection worker runtime — give it any file and it reports what it is and what's inside. Autonomously detected by the AWCP radar (it sends nothing to AWCP).

framework: LangGraph model: Ollama · qwen2.5:7b + qwen2.5vl:7b port: 8104 folder: agents/fileinspector_agent

What it is

This is not a chatbot — it is an autonomous worker in the AWCP sense: an "agent on a runtime" that the control plane observes. You hand it a file (drag-and-drop, or a typed path); it queues a goal, then executes it in multiple steps — classify the file, extract its contents (or caption it, for images), and summarise what it found — then routes the report through its own governed-write path (risk-tiered, with operator approval for high-risk external writes). It is a LangGraph runtime driving a local Ollama model (free, no API keys): a deterministic classify/extract pipeline plus a single grounded LLM summary.

It handles text, images, PDFs, JSON, Word .docx, CSV, source code, and unknown binaries (sniffed by magic bytes). The output is a brief summary of the contents only — no technical metadata. Image understanding uses a local Ollama vision model (qwen2.5vl); text and documents are summarised by a strong local text model (qwen2.5). If the summary model is unavailable, a fallback shows the extracted content directly.

Architecture — how a file flows

Features

  • Any file in — drag-and-drop upload (POST /upload) or a typed path; the agent classifies images, PDFs, .docx, text/JSON/CSV/code, and unknown binaries.
  • Multimodal — images are described by a local Ollama vision model (Qwen2.5-VL by default).
  • Task queue + background worker — submit files; the worker processes them one at a time, autonomously.
  • Multi-step pipeline — classify_file, then read_document / describe_image, then a single grounded LLM summary of what the file is and contains.
  • Graceful fallback — if the summary model is unavailable, the file's extracted content is shown directly so you still see what's inside.
  • Self-governed writes — risk-tiered; high-risk external writes pause for operator approval. Enforced by the agent itself — no control plane required.
  • Decoupled & autonomously detected — sends nothing to AWCP; the radar discovers it by scanning the process (observe-only).
  • Free & local — runs on local Ollama models and local libraries; no API keys, no accounts.

How to run

  1. Start the model backend (free, local): 
    ollama serve
ollama pull qwen2.5:7b         # reading / summarising text & documents
ollama pull qwen2.5vl:7b       # image captioning
  2. Start this worker (self-bootstraps its venv on first run): 
    bash agents/fileinspector_agent/run.sh
  3. Open the console → http://localhost:8104, drop a file, and watch it run.

Or manage all agents from one place: python3 agents/control_panel.py → http://localhost:8099.

Governance & its relationship to AWCP

This agent is decoupled from the AWCP control plane — it sends nothing to AWCP. Governance is split in two:

  • Agent-local (enforced here): risk tiers + an operator approval step that pauses high-risk external_post writes in awaiting_approval until approved on this agent's own POST /tasks/{id}/approve endpoint.
  • AWCP-side (observe-only): the radar detects this process autonomously by scanning it and lists it (as quarantined, since the agent declares no control hooks).
Model note. Reading/extraction is deterministic Python (Pillow / pypdf / python-docx), so it never depends on the model — the model only writes the short summary of the contents. If the text model is unavailable, a fallback shows the extracted content directly. Image descriptions need a vision model (qwen2.5vl by default; llama3.2-vision, minicpm-v, or llava also work) — without one, images can't be described.

Part of the AWCP agent runtimes. The queue, worker, governance, approval flow and UI are shared in agents/fileinspector_agent/awcp_kit.py (plus one additive /upload + drag-drop feature, gated by accepts_files); this agent supplies the LangGraph framework agent and its file-inspection tools.
